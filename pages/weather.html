WeatherNow

Weather website, spring 2019

Objective

The goal was to create a website that would give a user all the needed information about the weather at the current moment.

Looking at existing platforms

Looking at the popular weather apps/websites helped to find the design pattern, it's strong and weak sides. Keeping the interface simple was the final decision after exploration.

Front-end

Because people mostly need current weather information ,main focus was mode on the first section - current weather state. Also Celsius to Fahrenheit button is ususlly hard to find. It was moved to the top right section to improve UX. But with a transparent background to reduce attention to it.

Wireframes. Content position exploration.

Mainly inspired by the weather IOS app style colours were inverted to achieve the best contrast with the moving background.

Back-end part

Google maps API was used to determine location of the device. After, latitude and longitude are put into openwether request. This allows the website work all around the world any time. Both these API sources are free and open, so no API access key was needed.

Thoughts

If there were more time the website would be able to predict wether in the future. Animation would be more smooth. Because there are so many weather states more icons and videos, weather descriptions could have been made.
